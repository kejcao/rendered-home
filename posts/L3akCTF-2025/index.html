Lowkey RSA

The code:

from Crypto.Util.number import *

def gen_primes(SIZE):
    p = random_prime(1 << (SIZE - 1), 1 << SIZE)
    while True:
        q = random_prime(1 << (SIZE - 1), 1 << SIZE)
        if p < q: 
            p, q = q, p
            if q < p < 2*q:
                break
    return p, q

nbits = 1024
flag = b"L3AK{<REDACTED>}"

R = RealField(nbits)
p, q = gen_primes(nbits//2)
N = p*q
phi = (p**4-1)*(q**4-1)
N_s = R(N**2)
N_ss = R(N**4)   
t = (2*N_ss-49*N_s + 2)/(4*N+170*N_s)

while True:
    d = randint(1, round(sqrt(t)) - 1)
    if gcd(phi-d, phi) == 1:
        break

e = inverse_mod(phi-d, phi)
c = pow(bytes_to_long(flag), e, N)
print(f"e = {e}\nN = {N}\nc = {c}")

This is a standard RSA system with $n=pq$ and $c\equiv m^e \pmod{n}$ except for one twist: The exponent $e$ is derived by a certain procedure. Specifically,

$$\phi = (p^4-1)(q^4-1)$$

$$e := (\phi-d)^{-1} \mod \phi$$

Empirically speaking, I found $d$ to be a random 1024-bit number. We would like to find the value of $\phi$ so we can invert $e$ and decrypt the flag. Let's analyze $e$ a bit further.

$$e\equiv (\phi-d)^{-1} \pmod\phi$$

$$e(\phi-d)\equiv 1 \pmod\phi$$

$$e\phi-ed\equiv 1 \pmod\phi$$

$$ed\equiv -1 \pmod\phi$$

Implying $ed = k\phi-1$ for some $k$. Furthermore, keep in mind that $1/d \approx 0$ and $\phi\approx (n^2-1)^2$. Therefore,

$$ed + 1= k\phi$$

$$e + 1/d = k\phi/d$$

$$e \approx k(n^2-1)^2/d$$

$$\frac{e}{(n^2-1)^2} \approx \frac{k}{d}$$

But we are given both $e$ and $n$, so the LHS is determined completely. We compute the convergents of the LHS to approximate the RHS, and we hope that one of these rational approximations is correct. Once we have $k,d$ we know $\phi = (ed+1)/k$. Consequently, we can invert $e$ and decrypt the flag.
