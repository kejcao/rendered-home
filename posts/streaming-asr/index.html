In _greedy_decode() the time_idx parameter is an index looping from 0 to out_len. out_len was set to 11, when the appropriate value is 2. By following out_len up the stack trace, we find that a return value encoded_len of cache_aware_stream_step() is responsible. Before returning, this function passes the values to streaming_post_process which importantly implements this logic:

if self.streaming_cfg.valid_out_len > 0 and (not keep_all_outputs or self.att_context_style == "regular"):
    encoded = encoded[:, :, : self.streaming_cfg.valid_out_len]
    encoded_len = torch.clamp(encoded_len, max=self.streaming_cfg.valid_out_len)

For the working model, self.streaming_cfg.valid_out_len is set to 2, while the broken has it set to 14. A solution is to add this line of code somewhere:

asr_model.encoder.streaming_cfg.valid_out_len = 2
